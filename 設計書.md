## 1. 目的とコンセプト

隼人CLUB の運営状況を、監督が一目で把握・実感できる経営ダッシュボード。
単なる売上管理ではなく、**「水谷哲也という人物のレガシーが、今この瞬間も社会に広がっていること」を可視化する**ことを最上位コンセプトとする。

### 設計方針
- **管理しやすく、編集しやすい** … 数値・文章・ランキングをすべて画面上で直接書き換え可能
- **期間で追える** … 今月/四半期/年初来/全期間 + 任意月 + 期間範囲検索で、時系列の実績を追跡
- **高級感・信頼感** … 白基調 × 赤（隼人CLUBブランド）× 金のアクセント
- **自己完結** … 特別なサーバー・DB不要。HTMLファイル単体で動作

## 5. データモデル

全データは JS 冒頭の **`DEFAULTS` オブジェクト**に集約（＝「今月」を基準とする単一の真実）。

```
DEFAULTS = {
  members:  { total, newThisMonth, retention, churn }
  region:   { labels[], data[] }
  age:      { labels[], male[], female[] }
  ai: {
    consultations, monthly, avgPerUser,
    satisfaction, satResponses,
    categories[{name,count}], philosophy[{name,count}]
  }
  social:   { eduEnrolled, lifePlans, expertConsult, marketUse, costSaved, donation }
  ranks:    [{name, note, count} × 6]
  ec:       { ecSales, ecOrders, ecAov, products[{name,count}] }
  video:    { videoViews, videoCount, videoAvg, list[{name,count}] }
  revenue:  { monthlySales, royalty, chairmanShare, arpu,      // chairmanShare の表示名は「監督分配」
              series:{sales[12],royalty[12],share[12]},
              ltv:{arpu,monthsAvg,grossMargin} }
  growth:   { members[12], aiRate[12], retention[12], eduRate[12] }
  impact:   [{ic, num, u, cap} × 6]
  voices:   [{body, who, tag, stars}]
}
```

> **命名メモ:** 監督分配のデータキーは既存コードとの互換のため内部識別子 `chairmanShare` のまま据え置き、画面上の表示ラベルのみ「監督分配」とする。

### KPIカードの共通形
`{ label, val, unit, delta }`（`delta` は前期比%、`lowerIsBetter:true` で解約率など「低いほど良い」判定）。

## 6. 期間・月・範囲フィルター（中核ロジック）

### 6.1 データの3階層

| 種別 | 値 | 保存 | 生成方法 |
|---|---|---|---|
| プリセット4種 | 今月 / 今四半期 / 年初来 / 全期間 | ✅ 永続・個別編集可 | 起動時に base から自動生成（`buildDefaultsAll`） |
| 月別（12ヶ月） | `M0`〜`M11` | ❌ 派生（非保存） | `computeMonth(idx)` |
| 期間範囲検索 | `R{s}-{e}` | ❌ 派生（非保存） | `computeRange(s,e)` |

`STORE = { 今月, 今四半期, 年初来, 全期間 }` が永続対象。`DATA` は現在表示中の1セットを指すポインタ。切替時に `DATA` を差し替えて `renderAll()`。

### 6.2 スケーリング2軸（`scaleData(c, ff, mf)`）

| 分類 | 対象 | 係数 |
|---|---|---|
| **フロー（活動量）** `ff` | 新規会員・AI相談・EC・動画・社会貢献の各KPI、各ランキング件数 | 期間の月数 or 活動量合計 |
| **会員規模（内訳）** `mf` | 地域別・年代男女・階級分布 | 会員数比 |
| **スナップショット/累計** | 総会員数・継続率・解約率・単価・満足度・LTV・HAYATO IMPACT | ×1（非スケール） |

### 6.3 係数の決定

| 表示 | `ff` | `mf` |
|---|---|---|
| 今月 | 1 | 1 |
| 今四半期 | 3 | 1 |
| 年初来 | 当月までの月数（例:7月=7） | 1 |
| 全期間 | 12 | 1 |
| 月別 `Mn` | `members[n]/members[今月]` | 同左 |
| 範囲 `R s-e` | `Σ members[m]/members[今月]` | `members[e]/members[今月]` |

### 6.4 実測値による上書き（月/範囲）
`computeRange` / `computeMonth` はスケール後、月次トレンド配列の実測値で以下を上書き：
- 売上/ロイヤリティ/分配 = `revenue.series` の期間合計（月別は当月値）
- 総会員数 = 期末月の `growth.members[e]`
- 新規会員 = 期間の純増（`members[e] − members[s-1]`）
- 継続率 = `growth.retention[e]`

### 6.5 期間表示
`periodInfo(v)` が選択値から「期間名（タグ）」と「いつ〜いつ」を算出し、ヘッダーのピルと各セクション見出しに表示。基準日は先頭定数 `NOW`（現状 2026-07-04 固定。実運用では `new Date()` に変更可）。

## 7. 編集・永続化機能

### 7.1 編集モード
「✎ 編集モード」ONで、`[data-bind]` 属性を持つ全要素が `contenteditable` 化。
- **インライン編集**：KPI数値・文章・ランキング名・階級名/説明・HAYATO IMPACT・会員の声（`data-bind` でデータパスに直結）
- **数値エディタパネル**：グラフ（地域/年代/収益推移/会員数推移）の各数値を入力欄で編集（`data-editor` パネル）
- 変更は `focusout`/`change` で即 `setBind()` → `save()` → `renderAll()`

### 7.2 保存・入出力
| 操作 | 挙動 |
|---|---|
| 自動保存 | 編集のたび `STORE` 全体を localStorage に保存 |
| ⬇ 書き出し | `STORE` を JSON ファイルでダウンロード |
| ⬆ 読み込み | JSON を読み込み `STORE` に反映 |
| ↺ 初期化 | localStorage 削除 → 初期値に戻す |
| 会員の声 | ＋追加 / 🗑削除（編集モード時・プリセット期間のみ） |

> 月別・範囲検索は派生ビューのため保存対象外（編集しても STORE を破壊しない安全設計）。派生ビューでの声の追加/削除はブロックし、プリセットへの切替を促す。

## 9. 主要関数リファレンス

| 関数 | 役割 |
|---|---|
| `buildDefaultsAll()` | 起動時に4プリセットを base から生成 |
| `scaleData(c, ff, mf)` | フロー係数・会員係数で数値を一括スケール |
| `computeRange(s,e)` / `computeMonth(i)` | 月/範囲の派生ビューを算出 |
| `periodInfo(v)` / `updatePeriod()` | 期間名・日付範囲の算出と表示更新 |
| `load()` / `save()` | localStorage 入出力（`STORE` 単位） |
| `getBind()` / `setBind()` | ドット記法パスで `DATA` を読み書き |
| `selectView(v)` / `renderAll()` | ビュー切替・全セクション再描画 |
| `kpiCard()` / `rankList()` | KPIカード / ランキングの描画 |
| `buildCharts()` | Chart.js 全グラフ構築 |
| `editorPanel()` | 編集用データパネル生成 |
| `bindEditable()` / `setEditableState()` | インライン編集の結線 |
| `buildScrollSpy()` | ナビのスクロール追従ハイライト |

## 11. 運用メモ

- ローカル確認：launch.json の `hayato-dashboard`（http://localhost:3472 ）
